
# IdeaGora

###  Descripción del Proyecto

**IdeaGora** es una plataforma web diseñada para conectar a estudiantes, docentes y autoridades de universidades del Ecuador, facilitando el acceso y la interacción con contenido académico en un solo lugar.

El sistema integra dos módulos principales:

-  Repositorio Académico: Permite subir, organizar y consultar documentos como tesis, investigaciones y archivos técnicos.
-  Sistema de Feedback Social: Un espacio tipo red social donde los usuarios pueden compartir ideas, comentarios y sugerencias.





##  Funcionalidades Principales

###  Repositorio de Documentos
- Subida de archivos (PDF, DOCX, etc.)
- Organización por categorías (tesis, informes, investigaciones)
- Búsqueda por título, autor o universidad
- Descarga de documentos

###  Sistema de Feedback
- Publicación de posts (ideas, comentarios, sugerencias)
- Sistema de likes y comentarios
- Feed dinámico
- Interacción entre usuarios

###  Gestión de Usuarios
- Registro e inicio de sesión
- Perfiles de usuario
- Edición de información personal

###  Notificaciones
- Alertas por interacciones (likes, comentarios)
- Actividad relevante dentro de la plataforma

## Como correr el proyecto

### 1) Backend

```bash
cd backend
npm install
npm run dev
```

### 2) Frontend

```bash
cd frontend
npm install
npm run dev
```

## Flujo que debes entender primero

1. El usuario entra desde el frontend.
2. El frontend llama al backend (`/api/...`).
3. El backend valida y consulta Supabase.
4. El backend responde al frontend.
5. El frontend actualiza la interfaz.

## Rutas principales de la app

- `/` inicio
- `/explorar` lista de proyectos
- `/categorias` redirige a explorar
- `/login` iniciar sesion
- `/register` registro
- `/perfil` perfil del usuario

## Nota para aprendizaje

Si estas empezando, revisa los archivos en este orden:

1. `frontend/src/App.jsx`
2. `frontend/src/context/AuthContext.jsx`
3. `backend/src/routes/*.js`
4. `backend/src/controllers/*.js`
5. `backend/src/services/*.js`
